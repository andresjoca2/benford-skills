# 00 - Hub Problemas y Workflows

## Propósito
Entender los dolores reales, workflows actuales y fricciones donde puede entrar el producto.

## Resumen actual
- El wedge operativo más claro hasta ahora no es toda la auditoría de punta a punta, sino la estructuración de documentos y la ejecución de pruebas de detalle sobre cuentas contables.
- La apuesta es ordenar documentos en una base de datos estructurada y usar un agent harness para ejecutar pruebas, hacer cruces de información y llegar a resultados finales con hallazgos.
- El foco inicial parece estar en pruebas de detalle sobre líneas específicas de los estados financieros, por ejemplo ingresos, gastos, inventarios, activos fijos y otras cuentas contables.
- Este wedge es atractivo porque aterriza la automatización en trabajo operativo real, repetible y central al proceso de auditoría.

## Hipótesis activas
- La estructuración documental y las pruebas de detalle pueden convertirse en un punto de entrada muy fuerte para automatizar auditoría.
- Resolver bien esta capa puede servir tanto para vender software a firmas como para operar auditoría directamente.
- Este wedge puede ofrecer una base de datos propietaria y reusable que aumente defensibilidad.
