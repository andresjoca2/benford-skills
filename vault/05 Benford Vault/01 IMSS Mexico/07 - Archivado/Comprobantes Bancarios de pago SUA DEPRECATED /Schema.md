# Schema

- - Objetivo del schema
    - Estandarizar la extracción de datos de comprobantes bancarios de pago SUA para uso en Excel o tabla estructurada.
    - Grain
      - 1 fila por comprobante bancario de pago SUA.
    - Tabla o JSON final
      - `sua_bank_payment_proofs`
    - Campos
      - `importe_pagado`
      - `fecha_pago`
      - `banco_emisor`
      - `referencia_visible`
      - `folio_sua`
      - `periodo_pago`
      - `linea_captura`
      - `razon_social`
      - `identificador_operacion`
      - `cuenta_cargo`
      - `hora_pago`
      - `entidad_receptora`
      - `canal`
      - `sucursal`
      - `source_file_name`
      - `source_variant`
    - Tipos de dato
      - `importe_pagado`: decimal(18,2)
      - `fecha_pago`: date
      - `banco_emisor`: string
      - `referencia_visible`: string
      - `folio_sua`: string
      - `periodo_pago`: string
      - `linea_captura`: string
      - `razon_social`: string
      - `identificador_operacion`: string
      - `cuenta_cargo`: string
      - `hora_pago`: string
      - `entidad_receptora`: string
      - `canal`: string
      - `sucursal`: string
      - `source_file_name`: string
      - `source_variant`: string
    - Llaves
      - `source_file_name + importe_pagado + fecha_pago + banco_emisor`
    - Campos calculados
      - 
    - Trazabilidad a origen
      - `importe_pagado`: total de pago / importe total visible en el comprobante
      - `fecha_pago`: fecha de pago / fecha de aplicación
      - `banco_emisor`: logotipo o razón social bancaria
      - `referencia_visible`: identificador visible útil
      - `folio_sua`: campo explícito cuando aparezca
      - `periodo_pago`: periodo de pago cuando exista
    - Relaciones con otras tablas
      - Puede cruzarse con ACUMSUA, DGE o vaciado de liquidaciones.
    - Ejemplo de output
      - `importe_pagado`: 1551113.68
      - `fecha_pago`: 2024-03-19
      - `banco_emisor`: Banorte
      - `referencia_visible`: 23216421
      - `folio_sua`: 120827
      - `periodo_pago`: 2024-02
