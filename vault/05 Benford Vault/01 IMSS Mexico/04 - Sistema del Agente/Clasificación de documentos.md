# Clasificación de documentos

## Propósito
Distinguir tipos de documento útiles para pensar inputs, soporte, outputs intermedios y entregables.

## Tipos de documento

### 1. Fuente raw
Documento o archivo que entra al sistema sin haber sido procesado por la prueba.

#### Ejemplos
- nómina original
- SUA original
- CFDI descargados
- balanza original

### 2. Documento soporte
Documento que no necesariamente se transforma, pero sirve para validar, justificar o evidenciar algo.

#### Ejemplos
- contrato colectivo
- contrato individual
- comprobantes de pago
- políticas por escrito

### 3. Output intermedio
Archivo o cédula que resulta de una limpieza, vaciado, cálculo o consolidación previa.

#### Ejemplos
- cédula de variables
- conciliación previa
- vaciado de pagos
- base transformada por registro patronal

### 4. Referencia normativa
Documento o fuente que no es dato operativo del cliente, pero define reglas, topes o criterios.

#### Ejemplos
- UMA
- porcentajes de seguros IMSS
- metodología IMSS
- reglas normativas

### 5. Entregable final
Documento que forma parte del resultado final de la auditoría.

#### Ejemplos
- plantilla / Excel del portal IMSS
- PDF final del cliente

## Regla práctica
Cuando se documente una prueba, conviene identificar cada documento usado según una de estas categorías para entender mejor:
- de dónde sale el dato
- qué se transforma
- qué solo soporta
- qué termina en el entregable final
