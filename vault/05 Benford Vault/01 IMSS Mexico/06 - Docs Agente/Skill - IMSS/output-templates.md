# Output templates for imss-pruebas-extraccion-conocimiento

## 1. SOP por auditor
Nombre sugerido:
`SOP - <Nombre del auditor>.md`

Estructura mínima:
- Objetivo de la prueba
- Cuándo aplica esta forma de ejecutar la prueba
- Documentos y archivos necesarios
- Dónde está la data relevante dentro de esos documentos
- Variables exactas que se extraen
- Proceso paso a paso
- Validaciones clave
- Casos especiales y bifurcaciones
- Output final de la prueba
- Relación con plantilla fuente / atestiguamientos / cédulas / otras pruebas
- Errores comunes o alertas

Reglas:
- Escribir para una persona de 20 años
- No asumir conocimiento experto
- No comparar con otros auditores
- Reflejar solo la versión operativa del auditor extraído

## 2. Cambios a Obsidian
Nombre sugerido:
`Cambios a Obsidian - prueba <x> - <auditor>.md`

Contenido mínimo:
- Información nueva identificada
- Documentos nuevos o ejemplos reales que deben agregarse
- Variables nuevas o mejor aterrizadas
- Relaciones prueba ↔ documento
- Relaciones prueba ↔ plantilla fuente
- Relaciones prueba ↔ atestiguamientos / cédulas
- Relaciones prueba ↔ otras pruebas
- Preguntas o zonas que todavía no conviene consolidar
- Archivos reales que deberían vivir dentro de la carpeta documental correspondiente en `02 - Documentos y Ejemplos/`

## 3. Preguntas abiertas
Nombre sugerido:
`Preguntas abiertas - prueba <x> - <auditor>.md`

Contenido mínimo:
- preguntas sobre objetivo
- preguntas sobre factores que cambian el proceso
- preguntas sobre documentos y ejemplos faltantes
- preguntas sobre ubicación de data y variables
- preguntas sobre pasos del proceso
- preguntas sobre outputs y trazabilidad
- para cada pregunta, explicar por qué importa y qué falta para cerrarla
