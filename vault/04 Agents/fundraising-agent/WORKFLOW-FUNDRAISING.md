# WORKFLOW-FUNDRAISING.md

## Input

The agent should receive structured fundraising or company-strategy context such as:
- fundraising objective
- round stage
- target check size
- geography
- investor profile
- current deck/narrative
- pipeline status
- market or PMF questions
- business model questions
- company design questions
- questions or requests from founders

## Main workflow

### 1. Interpret the strategic need
Extract:
- market question
- PMF question
- company design question
- capital question
- immediate execution need

### 2. Build strategic understanding
Produce structured thinking around:
- market structure
- customer segments
- pains and workflows
- competitors and substitutes
- PMF hypotheses
- company design implications

### 3. Translate strategy into capital logic
Help produce:
- fundraising rationale
- amount-to-raise scenarios
- milestone logic
- investor profile logic
- likely objections

### 4. Support investor targeting and messaging
Produce useful investor candidates and materials with:
- name
- type/stage fit
- geography fit
- why relevant
- source support when possible
- outreach framing
- diligence notes
- follow-up priorities

### 5. Structure output for operator use
Output should be easy to reuse for:
- strategy notes
- investor pipeline
- outreach prep
- meeting prep
- diligence follow-up
- internal fundraising memory
- Obsidian knowledge base

### 6. Learn from review
When outcomes come back later, preserve learnings such as:
- who replied
- what angles worked
- what objections repeated
- what investor types were strongest fits
- what strategic theses got stronger or weaker

## Quality bar

Good output is:
- relevant
- structured
- evidence-aware
- founder-useful
- explicit about uncertainty

Bad output is:
- vague
- overconfident
- generic investor lists with no rationale
- hard to reuse in the pipeline
