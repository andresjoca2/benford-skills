## Principios de naming

- Carpetas numeradas para dar orden estable.
- Cada carpeta temática tiene un `00 - Hub ...`.
- Las notas hijas deben tener nombres específicos, no genéricos.
- Evitar títulos ambiguos como `Notas`, `Ideas`, `Research 1`, `Pendientes`.

## Tipos de notas recomendadas

### Notas hub
Sirven para:
- orientar la navegación
- resumir tesis actuales
- enlazar notas hijas
- mostrar preguntas abiertas y siguientes pasos

### Notas de análisis
Sirven para:
- profundizar en un tema
- dejar hipótesis, evidencia e implicaciones

### Notas de decisión
Sirven para:
- registrar una decisión tomada
- explicar por qué se tomó
- dejar fecha, contexto y consecuencias

### Notas de pregunta abierta
Sirven para:
- capturar incertidumbres clave
- evitar que se pierdan
- convertirlas después en investigación o decisiones

## Reglas de linking

Toda nota importante debe enlazar al menos a:
- su hub temático
- una nota relacionada aguas arriba o aguas abajo

Toda nota hub debe enlazar:
- al `00 - Hub estratégico`
- a sus notas hijas principales
- a preguntas abiertas relevantes
- a decisiones tomadas relevantes

## Plantilla mínima por nota importante

- Resumen
- Contexto
- Hallazgo / hipótesis / decisión
- Evidencia
- Implicaciones
- Preguntas abiertas
- Siguiente paso

## Regla de calidad

No crear notas vacías por llenar volumen.
Cada nota debe existir porque:
- organiza mejor el sistema
- captura una idea reusable
- conecta una decisión
- preserva evidencia
- abre una línea de trabajo concreta
