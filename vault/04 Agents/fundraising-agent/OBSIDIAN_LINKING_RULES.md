# OBSIDIAN_LINKING_RULES.md

## Principio

La vault debe poder navegarse sin depender de memoria conversacional.
Cada nota importante debe estar conectada al sistema.

## Reglas básicas

1. Toda nota importante debe tener al menos un enlace a su hub temático.
2. Toda nota importante debe intentar enlazar una nota relacionada adicional.
3. Toda decisión importante debe enlazarse desde `10 - Decisiones Tomadas`.
4. Toda pregunta estratégica abierta debe enlazarse desde `09 - Preguntas Abiertas`.
5. Los hubs temáticos deben enlazar de regreso al `00 - Hub estratégico`.

## Regla práctica

Si una nota no se puede encontrar desde:
- el hub estratégico,
- un hub temático,
- preguntas abiertas,
- o decisiones tomadas,
entonces probablemente está huérfana y debe ligarse mejor.

## Qué evitar

- notas aisladas sin links
- duplicar hubs para el mismo tema
- usar nombres poco descriptivos
- mezclar hechos, hipótesis y decisiones sin marcarlo
